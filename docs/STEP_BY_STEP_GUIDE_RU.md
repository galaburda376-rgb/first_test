# Пошаговая инструкция: что делать дальше

Ниже — самый простой порядок действий, чтобы увидеть файлы, запустить проект и продолжить разработку.

## 1) Проверить, что ты в нужной папке проекта

```bash
cd /workspace/first_test
pwd
```

Ожидаемый путь: `/workspace/first_test`.

## 2) Проверить ветку и коммиты

```bash
git branch --show-current
git log --oneline -n 5
```

Сейчас рабочая ветка: `work`, а не `main`.

## 3) Запушить ветку в GitHub (чтобы файлы появились в UI)

```bash
git push -u origin work
```

После этого в GitHub:
- переключи ветку на `work`,
- либо создай Pull Request `work -> main`.

## 4) Установить Flutter (если не установлен)

Проверка:

```bash
flutter --version
```

Если команды нет, установи Flutter по официальной инструкции:
- https://docs.flutter.dev/get-started/install

После установки:

```bash
flutter doctor
```

Исправь пункты с ошибками (Android SDK/Xcode/лицензии и т.д.).

## 5) Запустить приложение

```bash
cd mobile_flutter
flutter pub get
flutter run
```

## 6) Проверить базовый сценарий MVP

1. Открой приложение.
2. Нажми `+`.
3. Нажми `Старт`, потом `Остановить`.
4. Нажми `Сохранить запись`.
5. Убедись, что запись появилась на главном экране.

## 7) Что делать следующим шагом в разработке

### Шаг 7.1 — Реальная запись аудио
- Подключить пакет записи.
- Сохранять путь к аудиофайлу.

### Шаг 7.2 — Локальное хранение
- Добавить базу (например, SQLite/Isar).
- Хранить: `id`, `createdAt`, `duration`, `filePath`, `transcript`.

### Шаг 7.3 — Транскрипция
- Интегрировать speech-to-text API.
- Отображать статус: `processing/done/error`.

## 8) Мини-чеклист на каждую задачу

Перед каждым коммитом:

```bash
git status
```

После изменений:

```bash
git add .
git commit -m "Короткое понятное описание"
git push
```

И убедись, что:
- функция проверена на устройстве,
- есть обработка ошибок,
- документация обновлена.
